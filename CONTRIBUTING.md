# Руководство по внесению вклада

Спасибо за интерес к проекту «ГдеМакс»! Мы приветствуем вклад от сообщества.

## Как помочь

1. **Сообщить об ошибке** — создайте issue с подробным описанием.
2. **Предложить улучшение** — обсудите идею в issue перед реализацией.
3. **Отправить pull request** — исправления, новые функции, документация.

## Процесс разработки

### Установка

```bash
git clone https://github.com/your-username/gdemaks.git
cd gdemaks
npm install
```

### Запуск

```bash
npm run dev:full
```

Приложение будет доступно на http://localhost:3000, а JSON‑сервер на http://localhost:3001.

### Стиль кода

- Используйте TypeScript с строгими типами.
- Следуйте правилам ESLint и Prettier (конфигурация уже в проекте).
- Компоненты пишите в функциональном стиле с хуками.
- Для стилей используйте Tailwind CSS (классы из дизайн‑системы).

### Структура проекта

```
src/
├── components/     # React‑компоненты
├── pages/         # Страницы (роутинг)
├── stores/        # Zustand‑сторы
├── services/      # Логика работы с API, модулями
├── types/         # TypeScript‑типы
└── utils/         # Вспомогательные функции
```

### Создание модулей (.gmod)

Модули — это JSON‑файлы с определением компонента, ценовых позиций и задач. Пример в `README.md`.

## Pull request

1. Создайте ветку от `main`: `git checkout -b feature/your-feature`.
2. Внесите изменения, соблюдая стиль.
3. Добавьте тесты, если возможно.
4. Убедитесь, что `npm run type-check` и `npm run build` проходят без ошибок.
5. Отправьте PR с описанием изменений.
