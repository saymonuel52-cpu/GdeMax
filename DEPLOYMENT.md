# Развертывание на Vercel

Это руководство описывает процесс развертывания приложения "ГдеМакс" на платформе Vercel.

## Предварительные требования

1. Аккаунт на [Vercel](https://vercel.com)
2. Репозиторий проекта на GitHub, GitLab или Bitbucket
3. Учетные данные Supabase (если используется)

## Шаги развертывания

### 1. Подготовка репозитория

Убедитесь, что в корне проекта присутствуют следующие файлы:

- `package.json` (с корректными скриптами сборки)
- `vite.config.ts`
- `vercel.json` (создан автоматически)
- `.env.example` (шаблон переменных окружения)

### 2. Импорт проекта в Vercel

1. Войдите в панель управления Vercel
2. Нажмите "Add New" → "Project"
3. Импортируйте репозиторий с вашим проектом
4. Vercel автоматически определит настройки сборки (Vite)

### 3. Настройка переменных окружения

В настройках проекта (Settings → Environment Variables) добавьте следующие переменные:

| Переменная | Значение | Описание |
|------------|----------|----------|
| `VITE_SUPABASE_URL` | URL вашего проекта Supabase | Получите из панели Supabase |
| `VITE_SUPABASE_ANON_KEY` | Anon Key вашего проекта Supabase | Получите из панели Supabase |

**Пример:**
```
VITE_SUPABASE_URL=https://abcdefghijklmnop.supabase.co
VITE_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

### 4. Настройка сборки

Vercel автоматически определит следующие настройки:
- **Build Command:** `npm run build`
- **Output Directory:** `dist`
- **Development Command:** `npm run dev`
- **Install Command:** `npm install`

При необходимости можно переопределить в `vercel.json`.

### 5. Домены и окружения

- **Production:** автоматически развертывается при пуше в ветку `main`
- **Preview:** автоматически создается для pull requests
- **Development:** можно настроить для других веток

### 6. Особенности проекта

#### Статический SPA
Приложение является одностраничным (SPA), поэтому все маршруты должны перенаправляться на `index.html`. Это уже настроено в `vercel.json`.

#### Supabase
Приложение использует Supabase для аутентификации и хранения данных. Убедитесь, что:
- CORS настройки Supabase разрешают домен Vercel
- В Supabase включена аутентификация по email

#### Мок-сервер (JSON Server)
В продакшене мок-сервер не используется. Для работы с реальными данными необходимо настроить Supabase или другой бэкенд.

### 7. Проверка развертывания

После развертывания:

1. Откройте URL вашего приложения
2. Проверьте, что интерфейс загружается
3. Проверьте аутентификацию (если настроена)
4. Убедитесь, что данные загружаются из Supabase

### 8. Мониторинг и логи

- **Logs:** доступны в панели Vercel (Deployments → выберите деплой → Logs)
- **Analytics:** встроенная аналитика Vercel
- **Speed Insights:** автоматический анализ производительности

## Устранение проблем

### Ошибка сборки
1. Проверьте логи сборки в Vercel
2. Убедитесь, что все зависимости установлены (`npm install`)
3. Проверьте совместимость версий Node.js (требуется 18+)

### Ошибки среды выполнения
1. Проверьте переменные окружения
2. Убедитесь, что Supabase доступен
3. Проверьте CORS настройки

### Маршрутизация (404)
1. Убедитесь, что `vercel.json` содержит правильные правила маршрутизации
2. Проверьте, что приложение использует HashRouter (если есть проблемы с SPA)
