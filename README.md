# ГдеМакс — Строительный ежедневник

Модульное веб-приложение для управления строительными объектами, сметами, складом, задачами и актами выполненных работ.

## 🚀 Быстрый старт

### Предварительные требования
- Node.js 18+
- npm или yarn

### Установка и запуск

```bash
# Клонирование репозитория (если нужно)
git clone <repository-url>
cd GdeMaks

# Установка зависимостей
npm install

# Запуск фронтенда (Vite) и мок-сервера (JSON Server)
npm run dev:full

# Или по отдельности:
npm run dev          # Фронтенд на http://localhost:3000
npm run serve        # Мок-API на http://localhost:3001
```

После запуска откройте [http://localhost:3000](http://localhost:3000) в браузере.

## 🧩 Модульная система

Приложение поддерживает загрузку пользовательских модулей в формате `.gmod` (JSON). Модуль может добавлять:
- Вкладки в интерфейс объекта
- Позиции в прайс-лист
- Задачи в календарь
- Чек-листы приёмки
- Собственные React-компоненты

### Пример файла модуля (`example_module.gmod`)

```json
{
  "id": "heated_floors",
  "name": "🌡️ Тёплые полы",
  "version": "1.0",
  "description": "Монтаж электрического тёплого пола",
  "category": "Инженерные системы",
  "icon": "🌡️",
  "priceItems": [
    {
      "name": "Укладка кабеля",
      "price": 500,
      "unit": "м²",
      "type": "work"
    },
    {
      "name": "Терморегулятор",
      "price": 1500,
      "unit": "шт",
      "type": "material"
    }
  ],
  "tasks": [
    {
      "title": "Укладка кабеля",
      "duration": 1,
      "dependsOn": []
    },
    {
      "title": "Заливка стяжки",
      "duration": 2,
      "dependsOn": ["Укладка кабеля"]
    }
  ],
  "checklist": ["Проверка сопротивления", "Фото укладки"],
  "componentCode": "function HeatedFloorsTab() { return <div>Специфический интерфейс</div>; }"
}
```

### Загрузка модуля
1. Перейдите в раздел «Магазин модулей»
2. Перетащите файл `.gmod` в область загрузки
3. Модуль появится в списке установленных и будет доступен для добавления на объекты

## 👥 Роли пользователей

- **Администратор** — полный доступ ко всем объектам и настройкам
- **Прораб** — доступ к своим объектам, управление задачами и складом
- **Мастер** — доступ к назначенным объектам, выполнение задач, отчётность
- **Заказчик** — только просмотр своих объектов, подписание актов

Демо-логин: на странице входа выберите роль, пароль не требуется.

## 🔧 Технологический стек

- **Frontend**: React 18, TypeScript, TailwindCSS, React Router v6
- **State Management**: Zustand (глобальное), React Query (кеширование)
- **Forms**: React Hook Form
- **Charts**: Recharts
- **Gantt**: Frappe Gantt (легковесная)
- **PDF**: jsPDF + html2canvas
- **PWA**: Service Worker, Web App Manifest
- **Mock API**: JSON Server (заглушка для облачной синхронизации)
- **Offline**: IndexedDB (через localForage), локальное состояние

## 📱 PWA и офлайн-режим

Приложение является Progressive Web App:
- Устанавливается на домашний экран
- Работает офлайн (кэширование статики и данных)
- Push-уведомления (заглушка)
- Фоновая синхронизация

## 🔌 Интеграции (заглушки)

- **1С**: экспорт сметы в CSV/XML
- **Telegram/WhatsApp**: отправка ссылок на объекты/акты
- **Google Calendar**: добавление задач в календарь
- **Облачная синхронизация**: мок-сервер с автоматической синхронизацией при наличии интернета
